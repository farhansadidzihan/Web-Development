History of Machine Learning

Machine learning (ML) has roots in the 1940s and 1950s, with early developments in pattern recognition and computer science. In the mid-1980s, ML became a separate field of study, with the introduction of decision tree algorithms and the Bayesian approach. The 1990s saw advancements in algorithms such as support vector machines (SVM) and artificial neural networks (ANN). The rise of big data and increased computational power in the 2000s led to the development of deep learning algorithms and increased interest in ML. Today, ML is a rapidly growing field with numerous applications in areas such as computer vision, natural language processing, and self-driving cars.

Revolution of Machine Learning

  • 1940s/1950s: Early developments in pattern recognition and computer science lay the foundation for ML.
  • Mid-1980s: ML becomes a separate field of study with the introduction of decision tree algorithms and Bayesian approach.
  • 1990s: Advancements in algorithms such as SVM and ANN.
  • 2000s: Big data and increased computational power drive the development of deep learning algorithms.
  • Today: ML is a rapidly growing field with numerous real-world applications, revolutionizing industries such as healthcare, finance, and transportation.

Types of Machine Learning

Supervised Learning

Supervised learning is a type of machine learning where the algorithm is trained using labeled data to make predictions or take actions based on input data. The algorithm uses the labeled data to learn the relationship between the input variables and the output variables and then generalizes this relationship to make predictions on new unseen data. In supervised learning, there is an expected output provided for each input example. Examples include classification and regression problems.Supervised learning is a type of machine learning where the algorithm is trained using labeled data to make predictions or take actions based on input data.   • Decision Tree

  • A Decision Tree is a Supervised Machine Learning algorithm which looks like an inverted tree, where in each node represents a predictor variable, the link between the nodes represents a Decision and each leaf node represents an outcome.
  • Structure of Decision Tree :- Root Node, Internal Nodes, Leaf/Terminal Nodes, Branches

  • It is based on the idea that the predictor variables in a Machine Learning model are independent of each other.

  • K-Nearest Neighbor (KNN) is a Supervised Learning algorithm that classifies a new data point into the target class, depending on the features of it's neighboring data points.

Unupervised Learning

Unsupervised learning is a type of machine learning where the algorithm is not given labeled data, and instead is asked to find patterns and relationships in the data without any guidance. The goal is to uncover the underlying structure of the data by clustering similar examples together, reducing the dimensionality of the data, or identifying anomaly and outliers. Common unsupervised learning algorithms include K-means clustering, Principal Component Analysis (PCA), and Autoencoders.

    Unsupervised Learning Algorithm

  • K-Means Clustering

  • K-means is an unsupervised learning algorithm for clustering, which means it is used to find groups of similar objects within a large data set. The idea behind K-means is to partition the data into K distinct clusters based on the similarity of their features. Each cluster is represented by its mean, also known as the centroid, and each object is assigned to the cluster with the closest centroid.

Reinforcement Learning

Reinforcement Learning (RL) is a type of machine learning where an agent learns to make decisions by interacting with its environment to maximize a reward signal. In RL, the learning process is modeled as a sequence of decisions, with each decision determined by a policy that maps the current state of the environment to an action to be taken. The agent receives feedback in the form of a reward signal that indicates how well the action it took led to the desired outcome. Over time, the agent learns to refine its policy so as to achieve the highest cumulative reward. Reinforcement learning has been applied to a wide range of problems, including game playing, robotics, and autonomous vehicles.

    Reinforcement Learning Terminologies

  • Agent : The Reinforcement Learning algorithm that learns from the trial and error.
  • Environment : The world through which the agent moves.
  • Action : All the possible steps that the agent can take.
  • State : Current condition returned by the environment.
  • Reward : An instant return from the environment to apprise the last action.
  • Policy : The approch that the agent uses to determine the next action based on the current state.
  • Value : The expected long term return with discount, as opposed to the short term reward.
  • Action-Value : This similar to Value, except, it takes an extra parameter, the current action.

    Reinforcement Learning Concepts

  • Reward Maximization

    Reward Maximization theory states that, a Reinforcement Learning agent must be trained in such a way that, he takes the best action so that the reward is maximum.

  • Exploration

    Exploration is about exploring and capturing more information about an environment.

  • Exploitaton

    Exploitaton is about using the already known exploited information to heighten the rewards.

  • Markov's Decision Process

    The mathematical approach for mapping a solution in Reinforcement Learning is called Markov's Decision Process (MDP).

    Reinforcement Learning Algorithm

  • Q-Learning

  • In Q-Learning, the algorithm attempts to learn the maximum expected reward for each action in each state, represented by a value called the Q-value. The Q-value for each action-state pair is updated after each interaction with the environment, using the observed reward and the estimated maximum expected reward for the subsequent state. The algorithm updates the Q-values using the Bellman equation, which provides a mathematical relationship between the current and future rewards.
  • The following equation is used to represent a Q-learning Formulae : Q(s, a) = Q(s, a) + α * (r + γ * max_a(Q(s', a')) - Q(s, a))

    Limitation of Machine Learning

  • Data quality

    Machine learning models are only as good as the data they are trained on. If the data is biased, incomplete, or of low quality, the model's performance will suffer.

  • Overftting

    Overfitting occurs when a model is too complex and fits the training data too well, but fails to generalize to new, unseen data. This is a common problem in machine learning, and can be addressed through techniques such as cross-validation and regularization.

  • Lack of interpretability

    Many machine learning models, particularly those in deep learning, are considered black boxes. This means that it can be difficult to understand how the model arrived at its predictions, which can be a barrier to its adoption in certain fields where interpretability is important, such as healthcare or finance.

  • Computational cost

    Some machine learning algorithms are computationally expensive, requiring significant amounts of memory and processing power. This can be a challenge when working with large datasets or in real-time applications.

  • Human bias

    If the data used to train a machine learning model reflects human biases, the model may reinforce and even amplify these biases. It is important to be aware of and address potential biases in the data to ensure that the model's predictions are fair and unbiased.
